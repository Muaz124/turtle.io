Elastic node.js web server / API server

turtle.io was created to reduce resource requirements, and simplify creation & deployment of modern web applications & API facades. Unlike other web servers, turtle.io is extremely easy to configure.

  • Middleware

    Connect middleware allows you to create custom applications easily.

  • Virtual Hosts

    Designed to host many websites and APIs from a single server.

  • Simplify your stack

    turtle.io can replace Apache, NGINX, etc., in your application stack.

Configuration

Configuration values can be set by passing an Object to start().

auth

Object

[Optional] HTTP Authentication configuration, key is the hostname & value is the parameters. The following are valid:

  • authRealm - Authentication realm.
  • authFile - File where user details are stored in format {user:pass} or {user:passHash} for basic access and {user:realm:passHash} for digest access.
  • authList - List where user details are stored in format {user:pass} or {user:passHash} for basic access and {user:realm:passHash} for digest access, ignored if authFile is specified.
  • authType - Type of authentication, may be basic or digest, optional, default is basic.
  • algorithm - Algorithm that will be used for authentication, may be MD5 or MD5-sess, optional, default is MD5. Only for digest authType.

cache

Number (1000)

Size of LRU cache for Etag validation.

default

String

[Required] Default hostname to handle requests which are not specified within vhosts; must be a valid entry within vhosts.

headers

Object

Response headers. CORS is enabled by default.

index

Array

Files to look for when accessing a directory resource.

logs

Object

Logging configuration.

  • format - Common Log Format string of tokens, defaulting to standard Virtual Host format (%v %h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-agent}i\")
  • level - Minimum Common Log Level which is emitted to stdout ("info")
  • stdout - Override & disable stdout emitting by setting to false (true)
  • syslog - Override & disable syslog emitting by setting to false (true)
  • time - Format for the date/time portion of a log message. (D/MMM/YYYY:HH:mm:ss ZZ)

pages

String (null)

Directory relative to root which has files named for HTTP status codes, to be served upon error, e.g. 404.htm.

port

Number (8000)

Port the server will listen on.

proxy

Object

Proxy configuration.

  • rewrite - Content-Type header values to apply URL rewrites to. (Array)
  • maxConnections - Maximum number of outbound connections for reverse proxies. (1000)

root

String (./sites)

Relative path to the web root directory.

session

Object

[Optional] Session configuration.

  • id - Session cookie name (sid)
  • valid - Semantic representation of how long a session is valid ("15m")
  • gc - Milliseconds between garbage collection of expired sessions (60000)

ssl

Object

[Optional] SSL configuration.

  • cert - Path to certificate file
  • key - Path to key file

uid

Number (null)

[Optional] UID the server runs as.

vhosts

Object

[Required] Virtual hosts the server will respond for, key is the hostname & value is the directory relative to root.
